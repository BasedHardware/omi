================================================================================
KNOWN ISSUES & SOLUTIONS
================================================================================

Issue 1: Apostrophe Escaping in Bash
-------------------------------------
Problem: Bash scripts fail when translations contain apostrophes
Solution: Use '\'' pattern (close quote, escaped apostrophe, open quote)

Example:
  CORRECT: jq '. + {"key": "l'\''application"}' app_fr.arb
  WRONG:   jq '. + {"key": "l\'application"}' app_fr.arb

Affected Languages: French, Italian, Catalan, Ukrainian

Issue 2: Const Widget Constraints
----------------------------------
Problem: Const widgets cannot use dynamic l10n values
Solution: Remove 'const' keyword from Text/widget constructors

Example:
  Before: const Text('Hello', ...)
  After:  Text(context.l10n.hello, ...)

Issue 3: BuildContext Access in Helper Methods
-----------------------------------------------
Problem: Helper methods need context.l10n access
Solution: Add BuildContext parameter to method signature

Example:
  Before: Widget _buildItem({required String text}) { }
  After:  Widget _buildItem(BuildContext context, {required String text}) { }

Issue 4: Permission Warnings (Non-Critical)
-------------------------------------------
Problem: "mv: set owner/group (was: 501/0): Operation not permitted"
Solution: Add "2>/dev/null" to mv commands (warnings are harmless)

Issue 5: Ukrainian Apostrophes
-------------------------------
Problem: Ukrainian apostrophe character breaks jq parsing
Solution: Remove apostrophe from Ukrainian translation or use alternative wording

Example: "Від'єднано" → "Відєднано"
