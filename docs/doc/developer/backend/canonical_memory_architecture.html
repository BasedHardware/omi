Stages (plain English)

1. Capture

Conversation/chat/OCR/manual writers call MemoryService. Canonical users get hash-idempotent rows in memory_items with evidence and TTL.

process_conversation.py:460 · product_memory.py:93

2. Maintenance pass

Per whitelisted user each hour (when enabled): TTL audit, then consolidation, then promotion — in that order.

run_canonical_short_term_maintenance · short_term_promotion.py:472

3. LLM consolidation

Code gathers candidates; one batched LLM call is the sole decider. Applies patches atomically; partial supersede failures block the watermark and defer promotion.

apply_consolidation_decision :618 · _should_advance_consolidation_watermark :411

4. Promotion

Batch-or-daily gate (25 items or 24h). Flips layer on the same row. Syncs vector, keyword index, and KG extraction.

5. Read / search

Default-visible short+long-term. Search = Typesense keywords + Pinecone vectors, RRF reranked (long-term active only).

Known gaps (review these)

  • Fast-track bypass — MEMORY_CANONICAL_PROMOTION_FAST_TRACK_ENABLED lets user_asserted items skip batch/daily gate (short_term_promotion.py:383)
  • LLM invoke exceptions — parse failures are handled; raw invoke errors may bubble uncaught (canonical_consolidation.py:374)
  • Corroboration re-bump — repeat consolidation on same short-term duplicate may increment count again (canonical_consolidation.py:705)
  • review_queue cascade — not fully wired on conversation/account delete (domain_model.md matrix)
  • Cron off by default — needs MEMORY_CANONICAL_PROMOTION_CRON_ENABLED=true
