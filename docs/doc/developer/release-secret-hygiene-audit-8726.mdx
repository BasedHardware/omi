## Scope

This audit covers public-safe release secret hygiene after PR #8717. It does not
use production credentials, inspect private release artifacts, run release or
deploy actions, or verify maintainer-only rotation evidence.

## Covered By PR #8717

- Public mobile client config is allowlisted in `app/config/client_env_policy.yaml`.
- `scripts/create-public-client-env.sh` emits only allowlisted `PUBLIC_*` values.
- `scripts/check-public-client-secrets.py` validates app env templates, generated
  env files, Codemagic public config generation, selected public Docker build
  args, direct provider domains, and duplicate workflow keys.
- `scripts/scan-public-artifact-secrets.py` extracts common public artifacts and
  fails on server-only variable names, current CI secret values, private key
  material, service-account files, and signing material.
- Codemagic mobile and desktop public artifacts run the source/env scanner before
  packaging and the artifact scanner after build.
- GitHub lint and pre-push run the public-client boundary check when relevant
  files change.

## Remaining Gaps Addressed For #8726

- Release workflow log hygiene now flags shell output commands that reference
  server-only env values, including dangerous `echo SECRET=$SECRET >> .env`
  patterns.
- `set -x` is now rejected when the traced block touches server-only env refs.
- Public Dockerfile scanning now covers broader `web/**`, `plugins/**`,
  `mcp/**`, and `desktop/**` Docker paths instead of only the first #8717 public
  web/plugin image set.
- Web public build arg allowlist now includes the existing public admin/app args
  `NEXT_PUBLIC_PLUGINS_APP_ID` and `NEXT_PUBLIC_OMI_API_URL`.
- `scripts/test-public-secret-scanners.py` exercises these checks with fake
  sentinel secrets and verifies artifact scans catch fake sentinel leakage in a
  zipped build-log fixture.

## Intentionally Out Of Scope

- Private maintainer evidence for actual key rotation and provider-side
  restrictions belongs in the private incident evidence log.
- Real release artifacts that may contain private data require a maintainer to
  provide a secure path and are not needed for the public scanner tests.
- Production release, deploy, key rotation, and provider console mutations are
  not part of this audit.
