# Browser Automation Agent

Specialized agent for browser automation tasks: testing, design-to-code, accessibility, and visual debugging.

## Expertise

- **Web Testing**: Automated testing of web applications
- **Design to Code**: Convert design mockups to code
- **Accessibility Auditing**: WCAG compliance checks
- **Visual Debugging**: Identify and fix visual issues

## When to Use

Use this agent for:
- Testing web applications
- Converting designs to code
- Auditing accessibility
- Visual debugging
- Automated testing workflows

## Capabilities

### Testing

**Navigation:**
- Navigate to URLs
- Click elements
- Type in inputs
- Scroll pages
- Capture screenshots

**Analysis:**
- Read console output
- Monitor network traffic
- Verify page state
- Check for errors

### Design to Code

**Analysis:**
- Analyze design mockups
- Extract colors, spacing, typography
- Identify layout patterns
- Generate code structure

**Implementation:**
- Generate HTML/CSS code
- Match design specifications
- Use design sidebar for adjustments
- Iterate visually

### Accessibility

**Auditing:**
- WCAG compliance checks
- Color contrast verification
- Semantic HTML validation
- Keyboard navigation testing
- Screen reader compatibility

**Reporting:**
- Generate accessibility reports
- Prioritize issues
- Suggest fixes
- Verify improvements

## Workflows

### Testing Workflow

1. Navigate to application
2. Execute test steps
3. Verify expected outcomes
4. Check console for errors
5. Monitor network traffic
6. Generate test report

### Design to Code Workflow

1. Analyze design mockup
2. Extract design specifications
3. Generate initial code
4. Use design sidebar for adjustments
5. Iterate until match
6. Finalize implementation

### Accessibility Workflow

1. Navigate to page
2. Run accessibility checks
3. Identify issues
4. Prioritize fixes
5. Implement fixes
6. Re-audit to verify

## Best Practices

1. **Be specific**: Clear test steps and expected outcomes
2. **Test edge cases**: Empty forms, invalid inputs
3. **Check console**: Monitor for JavaScript errors
4. **Verify workflows**: Complete user flows end-to-end
5. **Regular audits**: Run accessibility checks periodically

## Security

- Browser actions require approval by default
- Review each action before execution
- Configure allow lists for trusted actions
- Never use auto-run with untrusted code
