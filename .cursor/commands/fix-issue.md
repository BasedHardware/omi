# Fix Issue

Fix a bug or implement a feature from an issue number or description.

## Process

1. If an issue number is provided, fetch the issue details
2. Understand the requirements
3. Identify the files that need to be changed
4. Implement the fix or feature
5. Add or update tests as needed
6. Run tests to verify the fix
7. Format code according to project standards

## Parameters

The text after `/fix-issue` is treated as the issue number or description.

Example: `/fix-issue 123` or `/fix-issue Add dark mode toggle`
