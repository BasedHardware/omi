## Summary

Fixes macOS: Device Settings page unreachable - cannot pair BLE device (issue #5917)

### Root Cause

After consolidating settings in commits 5e2fd7db4 and b5700ea74, the Device section was removed from the Settings sidebar entirely. This created two problems:

1. **Catch-22 navigation**: The `deviceStatusWidget` in the sidebar was only shown when `deviceProvider.isConnected || deviceProvider.pairedDevice != nil` — meaning users who haven't paired a device yet couldn't see the widget at all, so they couldn't navigate to device settings to pair one.

2. **Wrong navigation target**: Even when the widget was visible, clicking it posted `navigateToDeviceSettings` notification which navigated to Settings (index 9) but never set `selectedSettingsSection` to a device section — because no `.device` case existed in `SettingsSection`.

### Changes

1. **SettingsPage.swift**:
   - Added `.device` case to `SettingsSection` enum
   - Added `deviceSection` computed property that embeds `DeviceSettingsPage()`
   - Added `.device` case to the switch statement

2. **DesktopHomeView.swift**:
   - Fixed `navigateToDeviceSettings` handler to set `selectedSettingsSection = .device` before navigating to Settings

3. **SidebarView.swift**:
   - Removed conditional that hid `deviceStatusWidget` when no device paired — now always visible

4. **SettingsSidebar.swift**:
   - Added `.device` case to `icon` computed property in `SettingsSidebarItem`
   - Added Device search items (Device, Bluetooth, Firmware Update) for search functionality
